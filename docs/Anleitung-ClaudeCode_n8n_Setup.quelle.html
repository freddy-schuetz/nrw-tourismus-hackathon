NRW-TOURISMUS-HACKATHON · n8n + CLAUDE CODE

Von Null zur eigenen Automatisierung

Anleitung für Einsteiger:innen — vom Installieren bis zum ersten laufenden Workflow
Starter-Repo: github.com/freddy-schuetz/nrw-tourismus-hackathon

Du brauchst keine Programmier-Erfahrung. Du beschreibst in normalen Worten, was dir im Alltag Arbeit macht — und Claude baut die Automatisierung für dich. Diese Anleitung führt dich Schritt für Schritt hin. Rechne mit rund 20–30 Minuten, bis alles steht.

Die zwei Werkzeuge in einem Satz

n8n ist ein Baukasten für Abläufe: „Wenn X passiert, mach Y." — zusammengeklickt statt programmiert. Claude Code ist der KI-Assistent, der diesen Baukasten für dich bedient: Du sagst, was du willst, Claude baut, testet und dokumentiert es.

Was du vorher brauchst

  • Einen Laptop (Windows oder Mac) mit Internet
  • Ein Claude-Konto mit bezahltem Plan (Pro, Max, Team oder Enterprise) — der Code-Bereich ist in der Gratis-Version nicht enthalten
  • Eine E-Mail-Adresse für den kostenlosen n8n-Test-Zugang
  • Nur Windows: „Git" muss installiert sein (Schritt 1 erklärt es)
1

Claude Code installieren

  1. Gehe auf claude.com/download und lade die App für dein System herunter.
  2. Installationsdatei ausführen, App starten und mit deinem Claude-Konto anmelden.
  3. Oben in der App gibt es drei Bereiche: Chat, Cowork und Code. Klick auf Code — dort wird gebaut.
⚠ Nur Windows: Git zuerst installieren

Unter Windows startet der Code-Bereich nur, wenn das Hilfsprogramm Git vorhanden ist — Claude kann es sich also nicht selbst nachinstallieren. Installer: git-scm.com/downloads/win — mit den Standard-Einstellungen installieren (einfach durchklicken) und Claude danach einmal neu starten. Auf dem Mac ist Git meistens schon dabei.

Wenn beim Klick auf „Code" eine Upgrade-Aufforderung kommt

Dann fehlt der bezahlte Plan. Der Code-Bereich braucht Pro, Max, Team oder Enterprise (Übersicht: claude.com/pricing). Nach dem Upgrade die App einmal neu starten.

Gut zu wissen: Claude Code ist in der App bereits enthalten — du musst nichts zusätzlich installieren oder in einem schwarzen Terminal-Fenster tippen.

2

n8n-Zugang holen (kostenlos)

Damit Claude Workflows für dich bauen kann, braucht es einen n8n-Zugang. Der Test-Zugang ist kostenlos und in 3 Minuten fertig.

  1. Auf n8n.io registrieren und die kostenlose Cloud-Trial starten.
  2. Notiere deine Adresse — sie sieht so aus: https://deinname.app.n8n.cloud
  3. In n8n ins Menü: Settings → n8n API → Create API Key.
  4. Kopiere den Schlüssel sofort und leg ihn kurz zur Seite (z.B. in eine Notiz).
Der Schlüssel wird nur einmal angezeigt

Wenn du ihn verlierst, ist das kein Drama: einfach in n8n einen neuen erstellen. Behandle ihn wie ein Passwort — nicht weitergeben, nicht in Chats posten.

3

Den Starter einrichten lassen

Jetzt kommt der bequeme Teil: Claude richtet alles selbst ein.

  1. Lege auf deinem Rechner einen leeren Ordner an, z.B. Dokumente\Hackathon.
  2. In Claude im Code-Bereich: Local auswählen, dann auf „Select folder" klicken und genau diesen Ordner wählen.
  3. Schreib Claude nun diesen einen Satz:
Das schreibst du Richte mir https://github.com/freddy-schuetz/nrw-tourismus-hackathon ein.

Claude lädt den Starter herunter, richtet ihn ein und fragt dich nach deiner n8n-Adresse und dem Schlüssel aus Schritt 2. Beides einfach in den Chat schreiben.

Was du dabei siehst — und was du tun musst

Bestätigen. Claude fragt vor Änderungen um Erlaubnis und zeigt dir, was es tun will → auf „Accept" bzw. „Zulassen" klicken.
Evtl. eine Windows-Sicherheitsabfrage (wenn ein Hilfsprogramm nachinstalliert wird) — mit „Ja" bestätigen.
Zum Schluss die App einmal neu starten, damit die n8n-Verbindung und das Zusatzwissen geladen werden. Claude sagt dir das auch.

4

Kurz testen, ob alles verbunden ist

Das schreibst du Prüfe die n8n-Verbindung.
✓ Wenn Claude „OK" meldet, bist du startklar.

Kommt stattdessen ein Fehler, schau in die Tabelle „Wenn's mal klemmt" weiter hinten — meist ist es nur ein Tippfehler in der Adresse.

5

Deine Idee klären — der beste Start

Der häufigste Stolperstein ist nicht die Technik, sondern: „Was genau soll ich eigentlich bauen — und wie sage ich das?" Dafür gibt es im Starter eine eingebaute Hilfe.

Das schreibst du Hilf mir, meine Idee zu klären.

Claude stellt dir dann eine einfache Frage nach der anderen — in Alltagssprache, ohne Fachbegriffe: Was nervt dich im Alltag? Was passiert immer wieder gleich? Womit arbeitest du? Am Ende bekommst du:

  • einen Steckbrief deines Ablaufs (Auslöser → Schritte → Ergebnis),
  • ein Ablauf-Diagramm zum Anschauen,
  • und daraus baut Claude direkt weiter.
Zwei Varianten — such dir aus, was passt

Noch gar keine Idee? Sag es einfach — Claude macht mit dir eine kurze Bestandsaufnahme und schlägt passende Ideen vor. Zum Stöbern gibt es im Projekt außerdem die Liste docs/tourismus-ideen.md.
Schon einen fertigen Plan? Dann sag „grill mich" — Claude klopft deinen Plan kritisch Frage für Frage ab, bevor gebaut wird. Das bewahrt dich davor, dir für den Hackathon zu viel vorzunehmen.

6

Den ersten Workflow bauen lassen

Sag einfach in eigenen Worten, was du brauchst. Zum Ausprobieren z.B.:

Das schreibst du Bau mir einen Workflow: Ein Webhook empfängt einen Namen und antwortet mit einer freundlichen Begrüßung.

Das passiert dann automatisch — ohne dass du extra danach fragen musst:

7

Optional: eigene Oberfläche & online stellen

Viele Ideen brauchen zusätzlich eine Seite zum Anklicken — ein Formular, ein Dashboard, eine Karte. Auch das baut Claude:

Beispiele, die du sagen kannst Bau mir ein Formular, das meinen Workflow startet.
Bau mir ein Dashboard, das die Ergebnisse aus meinem Workflow anzeigt.

Anschauen

Sag „starte mein Frontend". Claude startet alles selbst und gibt dir einen Link wie http://localhost:3000 — draufklicken, fertig. localhost heißt: läuft auf deinem eigenen Laptop, ideal zum Bauen und für eine Demo am eigenen Bildschirm.

Für alle im Internet erreichbar machen

Für eine echte Web-Adresse (z.B. zur Präsentation) gibt es den kostenlosen Dienst Vercel:

  1. Einmalig selbst: auf vercel.com ein kostenloses Konto anlegen. Öffnet Claude danach ein Browser-Fenster zum Anmelden, kurz bestätigen.
  2. Dann sagst du: „stell mein Frontend online" — Claude erledigt den Rest und gibt dir die fertige öffentliche Adresse.

Dein n8n bleibt davon unberührt — es läuft ja schon in der Cloud. Und wenn du deine Arbeit sichern oder im Team teilen willst: „lade mein Projekt auf GitHub" (dafür brauchst du ein kostenloses GitHub-Konto).

Drei Tipps aus der Praxis

1. Klein anfangen. Ein Ablauf, der wirklich läuft, ist mehr wert als drei halbfertige.
2. In eigenen Worten reden. Du musst keine Fachbegriffe kennen — beschreib es so, wie du es einer Kollegin erklären würdest.
3. Nachfragen ist erlaubt. „Warum hast du das so gebaut?" oder „Erklär mir das nochmal einfacher" — Claude antwortet dir geduldig.
